list And forward_list

Last Updated: 14 july 2021


A list are sequence containers which are implemented like doubly-linked lists. Internally like in doubly linked list the association to each element of a link to the element preceding it and a link to the element following it.
List are very similar to forward_list ,the main difference being that forward_list objects are single-linked lists, and thus they can only be iterated forwards, in exchange for being somewhat smaller and more efficient.
In comparision to other sequence containers like array, vector and deque, lists perform generally better in inserting, extracting and moving elements in any position within the container for which an iterator has already been obtained.
The drawback of lists and forward_lists compared to these other sequence containers is that they lack direct access to the elements by their position.



Methods used in list are

  • begin() : Returns the iterator pointing to the first element in list.
  • end() : Returns the iterator to the end of the list.
  • rbegin() : Returns the reverse iterator pointing to the last the element.
  • rend() : Returns the reverse iterator pointing to the first element.
  • cbegin() :Returns a constant iterator to begining.
  • cend() : Returns a constant iterator to end.
  • crbegin() : Returns constant reverse iterator to end.
  • crend() : Returns constant reverse iterator to begining.
  • front() : To access the first element.
  • back() : To access the last element
  • empty() : To check whether container is empty.
  • size() : Returns the size.
  • max_size() : Returns the maximum size.
  • insert() : Insert the element at a specified position
  • assign() : assigns new element by replacing old.
  • push_front() : Insert element at the beginning .
  • pop_front() : deletes element form front.
  • emplace() : constructs and insert element
  • emplace_front() : constructs and inset element at beginning.
  • push_back() : Insert element at the end
  • pop_front() :Deletes element from front
  • emplace_back() : Construct and insert element at end.
  • erase() : erases element
  • remove() : remove element with specific value
  • clear() : clear the list
  • sort() : sort the list by default ascending
  • merge() : merge the lists
  • swap() : swap content of list
  • splice() : Transfer element from list to list
  • unique() : removes the duplicate values
  • reverse() : reverse the list
  • resize() : change the size


Implementation of list methods



A forward_list are implemented as singly-linked lists, so each of the elements they contain is stored in different and unrelated storage locations. The ordering is kept by the association to each element of a link to the next element in the sequence.
However,the main difference between a forward_list and a list is that forward list are implemented as sinlgy linked list which means they can only move forward as only the address of the next element is stored in association with the previous one , thus increasing the storage space required to store each element. So we can say forward_list objects are more efficient than list objects, although they can only be iterated forwards.
In comparision to other containers like array, vector and deque, forward_list perform generally better in inserting, extracting and moving elements in any position within the container.
The drawback of forward_lists and lists compared to these other sequence containers is that they lack direct access to the elements by their position.

  • before_begin() : Returns the iterator before beginning.

  • cend() : Returns a constant iterator to end.
  • cbefore_begin() : returns constant iterator before beginning
  • front() : To access the first element.
  • empty() : To check whether container is empty.
  • size() : Returns the size.
  • max_size() : Returns the maximum size.
  • insert_after() : Insert the element at a specified position.

  • emplace_after() : constructs and insert element

  • erase_after() : erases element
